Reconnaissance

Reconnaissance involves gathering information about the target system, such as domain names, IP addresses, and network infrastructure. It is divided into two parts: Active and Passive Reconnaissance.

Scanning (üîé)

Scanning involves identifying vulnerabilities in the target system using tools like Nessus, Nmap, and others to find weaknesses.

Exploitation (üí•)

In this phase, the attacker attempts to exploit vulnerabilities to gain unauthorized access to the system.

Post-Exploitation (üîí)

After gaining access, the attacker escalates privileges and maintains access. This phase may involve lateral movement within the network.

Reporting (üìä)

At the end of the test, a report is generated detailing vulnerabilities, exploitation methods, and recommendations for improvement.

Passive Reconnaissance (üëÄ)

Passive reconnaissance involves gathering information without directly interacting with the target. This may include examining public sources like WHOIS data and social media.

Active Reconnaissance (‚ö°)

Active reconnaissance involves directly engaging with the target by scanning the system or attempting to find vulnerabilities through direct probing.

Vulnerability Scanning (üîß)

Vulnerability scanning is the process of identifying weaknesses in the target system using automated scanning tools like Nessus or OpenVAS.

Network Mapping (üó∫Ô∏è)

Network mapping involves creating a map of the target's network, including identifying devices, services, and potential entry points.

Gain Access (üîì)

Gain access is the phase where an attacker uses identified vulnerabilities to gain unauthorized access to the target system.

Privilege Escalation (üöÄ)

Privilege escalation involves gaining higher access levels after initial access, enabling an attacker to perform more advanced operations on the target system.

Data Exfiltration (üì§)

Data exfiltration is the phase where sensitive data is extracted from the target system after gaining full access.

Cleanup (üßπ)

Cleanup involves removing traces of the attack, such as log entries or malware, to avoid detection by system administrators.

Generate Reports (üìù)

After testing, a detailed report is generated, documenting findings, methods, and specific vulnerabilities discovered during the test.

Provide Recommendations (üí°)

The recommendations phase involves providing the client with suggestions on how to mitigate the vulnerabilities discovered during the penetration test.
