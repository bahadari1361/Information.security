Monitoring & Logging Project

Central Bank of Afghanistan | Information Security Management

As part of the Information Security Management program at the Central Bank of Afghanistan, I designed and implemented a comprehensive monitoring and logging system that adhered to the ISO/IEC 27001 standards. This system was designed to continuously monitor critical infrastructure and applications, collect log data, and ensure rapid detection and response to potential security incidents.

Key Steps in Implementation

  • Define Security Monitoring Objectives: Identified the systems and processes that required monitoring, based on risk assessments and organizational security objectives.
  • Set up Logging Systems: Configured centralized logging for all systems using solutions like ELK Stack for log aggregation and analysis.
  • Develop Incident Response Plans: Created incident response procedures for handling and investigating security events identified through monitoring and logs.
  • Configure Alerts & Notifications: Set up automated alerts for critical system events using Splunk and Palo Alto Networks tools.
  • Ensure Compliance with ISO/IEC 27001: Ensured that all monitoring and logging practices complied with ISO/IEC 27001 requirements for auditing and continuous improvement.

Tools Used

  • ELK Stack: Used for centralized logging, real-time analysis, and visual representation of log data.
  • Splunk: Employed for security information and event management (SIEM) to monitor network activities and detect potential incidents.
  • Palo Alto Networks: Configured for firewall monitoring, log management, and incident detection.
  • Rsyslog: A logging tool used to forward logs from remote systems to a centralized server for analysis.
  • Nagios: Deployed for network monitoring, ensuring uptime and tracking system health.

Detailed Security Logs Overview

The log management system included categorization of logs into different categories such as system, application, security, and network logs. This allowed for efficient filtering and prioritization during incident investigations. The system also generated automated reports that complied with internal audit requirements and provided valuable insights into network behavior.

Compliance & Best Practices

  • ISO/IEC 27001 Compliance: The project adhered to the requirements for monitoring and logging as part of an Information Security Management System (ISMS), including proper log retention, encryption, and access controls.
  • Continuous Monitoring: Ensured 24/7 monitoring of critical systems to detect abnormal activities, failed login attempts, and potential intrusion attempts.
  • Data Integrity & Availability: Ensured log integrity by utilizing cryptographic hashes and limiting access to the logs to authorized personnel only.
