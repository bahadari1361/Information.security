Information Security Assessment Methodology

Central Bank of Afghanistan

Approach

The methodology adopted for this information security assessment includes a combination of documentation review, stakeholder interviews, vulnerability scanning, and risk assessments. The process follows a systematic approach to identify security weaknesses and compliance gaps.

Key Steps in the Methodology

Document Review: Review of existing security policies, procedures, and relevant documentation (e.g., business continuity plans, incident response protocols).

Interviews: Conduct interviews with key personnel to understand their roles and responsibilities and gather insights into the implementation of security measures.

Vulnerability Scanning: Perform technical vulnerability assessments using tools like Nessus, Nmap, and Acunetix.

Penetration Testing: If applicable, simulate attack scenarios to test the effectiveness of technical controls.

Compliance Evaluation: Evaluate compliance with ISO/IEC 27001 standards through a gap analysis.

Tools Used

The following tools were employed to carry out the security assessment:

  • Kali Linux: Used for penetration testing and ethical hacking simulations.
  • Nessus: For network vulnerability scanning.
  • Nmap: To assess network security and topology.
  • Wireshark: For packet sniffing and network traffic analysis.
  • Acunetix: For web application vulnerability scanning.
  • ntopng: For real-time network monitoring.

ISO 27001 Framework

The assessment follows the structure and control objectives defined by ISO/IEC 27001, focusing specifically on the following areas:

  • Information Security Governance and Risk Management
  • Access Control
  • Incident Response and Management
  • Security Operations and Monitoring
  • Business Continuity and Disaster Recovery
