Tools & Techniques Used in the Vulnerability Assessment

Comprehensive Tools and Techniques for Security Assessments at the Central Bank of Afghanistan

1. Kali Linux

Kali Linux is a powerful and versatile penetration testing distribution, providing a vast collection of tools for vulnerability assessment. I used Kali Linux as a comprehensive platform for conducting penetration tests and security audits at the Central Bank of Afghanistan.

  • Used Kali's pre-installed tools like Nmap for network scanning and OpenVAS for vulnerability scanning.
  • Utilized Metasploit for exploit testing and social engineering assessments.
  • Conducted thorough penetration testing to identify exploitable vulnerabilities in both internal and external network infrastructures.

2. Nessus

Nessus is a leading vulnerability scanner that helped in identifying known vulnerabilities, misconfigurations, and missing patches within the bank's network infrastructure. I used Nessus extensively to detect security weaknesses and recommend necessary patching actions.

  • Performed full network scans to identify vulnerabilities in servers, workstations, and other network devices.
  • Configured customized scan policies to address specific security requirements for the banking systems.
  • Generated detailed vulnerability reports with actionable remediation steps for the IT team.

3. Nmap

Nmap (Network Mapper) was used to conduct network discovery and security auditing. With Nmap, I performed detailed scanning to identify open ports, services, and possible vulnerabilities in network devices.

  • Used Nmap's advanced scripting engine (NSE) to check for common vulnerabilities and misconfigurations.
  • Utilized Nmap for OS fingerprinting to accurately identify systems running on the bank's network.
  • Conducted comprehensive network topology mapping to detect potential attack surfaces and misconfigurations.

4. Wireshark

Wireshark was employed for deep packet analysis, helping in identifying traffic patterns and security issues such as unencrypted sensitive data or unauthorized access attempts.

  • Monitored network traffic to identify any suspicious activity or communication over insecure protocols.
  • Captured packets to analyze data being transmitted across the network, identifying potential leaks or unauthorized access.
  • Used Wireshark in real-time to investigate incidents and verify security controls.

5. Acunetix

Acunetix is a web vulnerability scanner that helped in assessing web applications for security flaws such as SQL Injection, Cross-Site Scripting (XSS), and other common web vulnerabilities.

  • Ran automated scans of the bank’s web applications to identify common vulnerabilities.
  • Conducted detailed reports highlighting the severity of identified vulnerabilities and recommending remediations.
  • Used Acunetix to verify compliance with security standards and practices for the bank's digital services.

6. Metasploit

Metasploit was an essential tool for testing the exploitation of vulnerabilities found during the assessment. I used it for controlled penetration tests and vulnerability validation.

  • Tested discovered vulnerabilities by attempting controlled exploitation to simulate real-world attacks.
  • Developed and tested exploit payloads to verify if vulnerabilities could be remotely triggered.
  • Utilized Metasploit's database to track vulnerabilities, exploits, and results during the assessment process.

7. OWASP ZAP

The OWASP Zed Attack Proxy (ZAP) is an open-source web application security scanner that I used to find vulnerabilities such as Cross-Site Scripting (XSS) and SQL Injection within web applications.

  • Automated security scans to identify security flaws in the bank’s web applications.
  • Performed active scanning for vulnerabilities and manual testing for complex cases.
  • Integrated ZAP into the continuous security monitoring process to ensure ongoing protection for web applications.

8. Zenmap

Zenmap is the graphical user interface for Nmap, and I used it to visually analyze network scans and produce detailed reports, which helped the team prioritize vulnerabilities based on their severity.

  • Used Zenmap for network mapping and identifying network devices and hosts.
  • Visualized network scan results to identify attack vectors and potential entry points into the bank’s systems.
  • Assisted in creating a comprehensive vulnerability assessment report for internal and external stakeholders.
