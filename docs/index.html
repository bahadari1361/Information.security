Information Security Management - Central Bank of Afghanistan

Overview

As the Information Security Manager at the Central Bank of Afghanistan, I was responsible for designing, implementing, and managing a comprehensive set of security measures to protect the bank’s sensitive data, networks, and IT infrastructure. This portfolio outlines my work in developing security policies, business continuity plans, incident response strategies, conducting IT audits, penetration testing, and security assessments to ensure the organization’s resilience against evolving cyber threats. Additionally, I played a key role in the deployment of ISO 27001 to align the bank's security framework with international best practices.

1. Information Security Policy Development

  • Network Security Policy: Designed guidelines for firewalls, VPNs, and network segmentation.
  • Email Security Policy: Established encryption and anti-phishing protocols.
  • User Management and Password Policy: Implemented multi-factor authentication (MFA) and password management guidelines.
  • Data Disposal Policy: Defined secure methods for data destruction.
  • Internet Usage Policy: Instituted controls to block malicious sites.
  • FTP Server Policy: Established secure file transfer protocols.
  • Patch Management Policy: Developed a patch management framework for system updates.
  • Remote Desktop Policy: Defined secure access through VPNs and strong authentication.

2. Business Continuity Planning (BCP) Development

I led the creation of the Business Continuity Plan (BCP), ensuring the bank could maintain operations during and after disruptions:

  • Risk Assessments: Prioritized critical business functions for disaster recovery.
  • Disaster Recovery Strategies: Developed backup solutions, including off-site and cloud backups.
  • Crisis Management Procedures: Established communication strategies for stakeholders.

3. Incident Response Plan (IRP) Development

I developed a structured Incident Response Plan (IRP) to manage and recover from security incidents:

  • Incident Classification: Defined processes for identifying and classifying incidents.
  • Roles and Responsibilities: Created clear roles for the response team.
  • Communication Protocols: Developed communication strategies for internal and external reporting.
  • Post-Incident Analysis: Implemented procedures for post-mortem analysis and future improvements.

4. IT Audits

  • System Log Reviews: Identified security weaknesses through log reviews.
  • Regulatory Compliance: Ensured compliance with local and international standards like GDPR and ISO 27001.
  • Security Recommendations: Provided actionable improvements to enhance security posture.

5. Penetration Testing

  • External and Internal Penetration Testing: Simulated attacks to find network vulnerabilities.
  • Web Application Security Testing: Conducted assessments for SQL injection and XSS vulnerabilities using tools like Acunetix and OWASP ZAP.
  • Reporting and Remediation: Collaborated with teams to address and patch identified vulnerabilities.

6. Security Assessments

  • Vulnerability Scanning: Used tools like Nessus and Zenmap to identify system weaknesses.
  • Network Traffic Analysis: Analyzed network traffic with Wireshark to detect malicious activities.
  • Web Application Security: Assessed web applications with Acunetix and OWASP ZAP.
  • Social Engineering Tests: Ran phishing simulations and provided training to improve employee awareness.

7. Tools and Technologies Used

  • Penetration Testing Tools: Kali Linux, Nessus, Nmap, Metasploit, Zenmap.
  • Security Monitoring Tools: ntopng, Wireshark.
  • Vulnerability Scanning Tools: Acunetix, OWASP ZAP.
  • Threat Intelligence Tools: Maltego.

8. Compliance Frameworks

  • ISO 27001 Implementation: Established a structured ISMS to ensure confidentiality, integrity, and availability of sensitive information.
  • Continuous Improvement: Led regular reviews and audits to maintain the ISO 27001 certification and improve security measures.

Conclusion

As the Information Security Manager at the Central Bank of Afghanistan, I led the development and implementation of security strategies to protect the bank’s critical IT assets. From creating robust security policies to hands-on tasks like penetration testing and incident response planning, my work helped safeguard the bank against evolving cyber threats. The successful deployment of ISO 27001 further demonstrated my ability to align the bank’s security framework with global best practices, ensuring long-term resilience.
