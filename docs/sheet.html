Vulnerabilities Cheat Sheet

Common Vulnerabilities and Mitigation Strategies

Vulnerability Description Mitigation
SQL Injection Occurs when an attacker is able to manipulate SQL queries by inserting malicious SQL code into an input field. Use parameterized queries or prepared statements to prevent code injection. Sanitize user inputs and use ORM (Object Relational Mapping) tools.
Cross-Site Scripting (XSS) Involves injecting malicious scripts into web pages, allowing attackers to execute arbitrary code in a user’s browser. Sanitize user inputs and encode dynamic content to prevent script execution. Use Content Security Policy (CSP) headers to restrict allowed sources for scripts.
Cross-Site Request Forgery (CSRF) Exploits the trust a web application has in the user’s browser, forcing the user to perform actions without their consent. Implement anti-CSRF tokens to validate requests. Ensure all state-changing operations require authentication and authorization checks.
Insecure Deserialization Occurs when an application deserializes untrusted data, potentially allowing attackers to manipulate application behavior or execute malicious code. Use safe deserialization techniques and ensure that only trusted data is deserialized. Implement integrity checks such as hashes to verify data authenticity.
Broken Authentication Weak authentication mechanisms can allow attackers to impersonate users or bypass authentication controls, compromising system security. Implement multi-factor authentication (MFA), enforce strong password policies, and secure session management to mitigate authentication-related risks.
Security Misconfiguration Occurs when an application or server is incorrectly configured, leading to vulnerabilities like unnecessary services being exposed or default settings remaining in place. Conduct regular security audits and remove unnecessary services, ports, and features. Harden your server and application configurations based on security best practices.
Sensitive Data Exposure Occurs when sensitive data, such as passwords, credit card information, or personal data, is not adequately protected from unauthorized access. Encrypt sensitive data both at rest and in transit using strong algorithms. Implement strict access controls and avoid storing sensitive data unless absolutely necessary.
Insufficient Logging & Monitoring Lack of proper logging and monitoring prevents organizations from detecting or responding to potential security incidents in a timely manner. Implement centralized logging systems and real-time monitoring for critical events. Set up alerts for unusual activities or suspicious patterns.
Command Injection Occurs when user inputs are passed to system shell commands, allowing attackers to execute arbitrary commands on the server. Validate and sanitize all user inputs before passing them to system commands. Avoid direct use of user inputs in shell commands.
Path Traversal Allows attackers to access files outside of the designated directories by manipulating file paths. Validate and sanitize file paths, restrict access to known directories, and use allow-lists to limit the files users can access.
