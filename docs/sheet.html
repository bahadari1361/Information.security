Vulnerabilities Cheat Sheet

Common Vulnerabilities and Mitigations

Vulnerability Description Mitigation
SQL Injection Occurs when an attacker can insert or manipulate SQL queries in an application's input fields, compromising the database. Use parameterized queries and prepared statements. Sanitize and validate all user inputs.
Cross-Site Scripting (XSS) Involves injecting malicious scripts into web pages that can execute in the user's browser, leading to data theft or account compromise. Sanitize user inputs and encode output to prevent execution of malicious scripts. Use Content Security Policy (CSP).
Cross-Site Request Forgery (CSRF) Tricks a user into performing actions they didn't intend, often resulting in unauthorized changes to their account or settings. Use anti-CSRF tokens to validate requests. Ensure that sensitive actions require authentication and authorization checks.
Insecure Deserialization Occurs when untrusted data is deserialized into objects, allowing attackers to manipulate objects or execute arbitrary code. Use safe deserialization methods. Avoid deserialization of untrusted data and implement integrity checks.
Broken Authentication Weak authentication mechanisms that can allow attackers to impersonate users and gain unauthorized access to systems. Use strong, multifactor authentication. Protect session management and passwords with encryption and hashing.
Security Misconfiguration Occurs when an application or server is incorrectly configured, leaving it vulnerable to attacks like unauthorized access or information leaks. Regularly audit configurations and ensure that unnecessary services, features, or ports are disabled. Follow the principle of least privilege.
Sensitive Data Exposure Occurs when sensitive information like passwords or personal data is not properly protected and is exposed to unauthorized access. Use encryption for sensitive data at rest and in transit. Implement proper access control to restrict access to sensitive data.
Insufficient Logging & Monitoring Failure to log security-relevant events or monitor for suspicious activity that can lead to missed attacks or breaches. Implement comprehensive logging and real-time monitoring. Regularly review logs and establish alerting for unusual activities.
Command Injection Occurs when an attacker can execute arbitrary commands on the server by manipulating input fields that are passed to a system shell. Validate and sanitize all user inputs. Avoid passing user input directly to system shell commands.
Path Traversal Allows attackers to access files outside the web root directory by manipulating file paths in requests. Validate and sanitize file input paths. Ensure that the application can only access allowed directories.
